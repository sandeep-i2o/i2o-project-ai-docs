### 3.2 Identified Risks & Recommendations

| Risk Area | Description | Recommended Mitigation |
| :--- | :--- | :--- |
| **BigQuery Latency** | Querying large datasets for "on-the-fly" PDFs might exceed the 5s target if views are poorly indexed. | Ensure the `viz_bbx_filtered_detail` view is partitioned and queries use appropriate filters (sellerID, date) to minimize scan size. |
| **Data Migration** | The `investigation_report` table is new. Existing sellers won't have entries. | Implement a "lazy creation" strategy where a row is inserted in a `PENDING` state the first time a seller enters a specific swimlane. |
| **Concurrency** | Multiple AMs toggling status might lead to race conditions. | Use database-level optimistic locking or specific `PUT` endpoints that check existing state before transition. |
| **Asset Consistency** | JasperReports logo assets must match the approved branding across and within environments. | Centrally manage logo assets in a shared GCS bucket instead of embedding them in the WAR files. |
