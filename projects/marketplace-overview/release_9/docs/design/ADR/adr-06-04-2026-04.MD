# Use UI-Only Placeholder Metrics for Unsubscribed Marketplaces in release_9

## Status

Accepted

## Context

Release_9 requires unsubscribed marketplace cards on the right column, but a canonical source for marketplace metrics (`totalProducts`, `totalListings`, `totalResellers`) is not available.

Without a canonical source, any backend aggregation path would be speculative and could produce inconsistent values across environments. The requested directive is to avoid API/backend metric logic and default to `##` in UI only.

## Decision Drivers

- Avoid publishing speculative or incorrect client-facing metrics.
- Remove backend/API scope that depends on missing data contracts.
- Keep release_9 deliverable stable while preserving pilot/audit CTA workflows.

## Considered Options

- Option 1: Keep backend unsubscribed metrics API with inferred/temporary source.
- Option 2: Use frontend-only placeholders (`##`) with no unsubscribed metrics API/backend logic.
- Option 3: Block unsubscribed marketplace cards until canonical source is ready.

## Decision Outcome

Chosen option: "Option 2", because it prevents incorrect data exposure and aligns directly with the release directive while keeping the UI flow available.

### Positive Consequences

- No inaccurate unsubscribed metrics are shown to clients.
- Backend complexity is reduced for release_9 (no parser/cache/aggregation path).
- Implementation is deterministic and testable (`##` placeholder rendering only).

### Negative Consequences

- Users cannot compare unsubscribed marketplace volume metrics in release_9.
- Pain-level signal is deferred until canonical source is finalized.

## Pros and Cons of the Options

### Option 1

- Good: Preserves numeric metric experience in UI.
- Bad: Risks wrong values and unstable contracts.

### Option 2

- Good: Clear and safe behavior with zero speculative backend logic.
- Bad: Reduced analytical depth for unsubscribed cards.

### Option 3

- Good: Avoids placeholder UX.
- Bad: Removes an expected column and CTA context from release_9.

## More Information

- Affected architecture sections: 4.1, 6.1, 7.4, 8.2, 9.4, 10.5, 12.3, 13.1.
- Affected modules:
  - `frontendapplication-i2oretail`: unsubscribed cards render `##` placeholders.
  - `i2o-reseller`: no unsubscribed marketplace metrics endpoint/aggregation path in release_9.
