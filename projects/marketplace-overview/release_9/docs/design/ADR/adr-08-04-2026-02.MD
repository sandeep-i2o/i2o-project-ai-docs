# Apply `region='ALL'` Marketplace Mapping Across Platform Regions

## Status

Accepted

## Context

`brand_master.marketplace_ids` stores marketplace references used by `GET /marketplace-overview/config` to resolve subscription brands.  
Some marketplace records are defined as platform-level rows with `region='ALL'` (for example, `platform='Walmart', region='ALL'`).

Without explicit fallback logic, a brand mapped to `Walmart-ALL` is not propagated to `Walmart-US`, `Walmart-CA`, and other Walmart region subscriptions.

## Decision Drivers

- Keep brand mapping maintenance simple by allowing platform-wide mappings.
- Avoid forcing duplicate `marketplace_ids` entries per region when a platform-wide intent exists.
- Preserve exact-marketplace behavior while adding deterministic fallback semantics.

## Considered Options

- Option 1: Exact `marketplace_id` match only.
- Option 2: Exact `marketplace_id` match plus same-platform fallback via `marketplace.region='ALL'`.
- Option 3: Add a new normalized platform-level brand mapping table.

## Decision Outcome

Chosen option: "Option 2", because it reuses current schema, fixes real response gaps, and introduces minimal implementation risk.

### Positive Consequences

- Brands mapped to `platform + ALL` are automatically available to all regions of that platform.
- Data maintenance is reduced for broad platform coverage.
- Existing exact-marketplace mappings continue to work unchanged.

### Negative Consequences

- Brand lookup query becomes more complex (adds `EXISTS` fallback check).
- Potential overlap between explicit region mappings and `ALL` fallback must be handled by deduplication.

## Pros and Cons of the Options

### Option 1

- Good: Simplest query and behavior.
- Bad: Does not support platform-wide mappings already present in marketplace data.

### Option 2

- Good: Correct platform-wide behavior without schema change.
- Bad: Slightly higher query cost due to fallback check.

### Option 3

- Good: Cleanly normalized long-term model.
- Bad: Requires migration, backfill, and rollout not justified for release_9 scope.

## More Information

- Impacted architecture sections: 4.1, 6.1, 7.1, 8.1, 15, 18.
- Impacted API: `GET /marketplace-overview/config`.
- Impacted module: `i2o-reseller` (`MarketplaceOverviewRepository`, `MarketplaceOverviewConfigService`).
- Rule summary: include a brand when `subscription.marketplace_id` matches directly, or when any mapped marketplace row has same `platform` and `region='ALL'`.
