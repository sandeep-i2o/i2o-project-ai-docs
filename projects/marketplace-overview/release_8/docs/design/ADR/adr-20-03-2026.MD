# ADR 20-03-2026: Frontend Architecture Alignment (March 2026 Update)

## Status
Proposed (2026-03-20)

## Context
The project `marketplace-overview` requires an architecture update to align with the latest `SUMMARY-frontendapplication-i2oretail.md` (March 2026). The update is restricted to frontend-only changes as per user instructions. Key alignment points include framework versions, grid library licensing, testing framework enhancements, and module wiring patterns.

## Decision
1.  **Framework Alignment**: Update baseline versions to Angular 15.2.10, TypeScript 4.9.5, and Material 15.2.9.
2.  **Grid Strategy**: Explicitly switch from "AG Grid Community" to the internal **Common data-grid** component for the Marketplace Table view, as specified in the frontend summary.
3.  **Testing Strategy**: Supplement the existing testing strategy (Protractor/Jasmine) with **Playwright (^1.58.x)** to match the current enterprise standard.
4.  **Module Wiring**: Ensure all reusable components from `modules/common` are wired via **`CoreModule`** rather than a standalone `common.module.ts`, removing a potential architectural inconsistency.
5.  **Constants Strategy**: Formalize the use of feature-level `*.constants.ts` (e.g., `marketplace-overview.constants.ts`) for module-specific literals, reserving `global-constants.ts` for app-wide cross-cutting concerns.

## Technical Details
- **Angular**: 15.2.10
- **TypeScript**: 4.9.5
- **Testing**: Jasmine/Karma + Playwright ^1.58.x
- **Grid**: Common data-grid
- **Icons/Styles**: Angular Material 15.2.9, Bootstrap 5.2.0

## Consequences
- **Positive**: Frontend architecture perfectly matches the repository's current state and standards.
- **Positive**: E2E testing benefits from modern Playwright capabilities (video capture, trace viewer).
- **Neutral**: The underlying data-grid component abstracts grid complexities.
- **Negative**: None identified; this is an alignment update.
