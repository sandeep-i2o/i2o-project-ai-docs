# Story: AG Grid Enterprise Setup (Export & Paging)

## Implementation Scope
Render massive tabular data payload (10K+ SKU metrics) on the `frontendapplication-i2oretail` SPA without locking user DOM or exceeding visual constraints. Native bindings for AG Grid Enterprise virtualization handling infinite scroll and dynamic client/server processing combinations must be connected to the `SalesInsightsStateService`. This story introduces the explicit CSV Background Trigger (`POST /v1/sales-insights/export`) hook into the UI.

## Acceptance Criteria
- [ ] Connect `sales-data-grid.component.ts` via Enterprise Module bindings enforcing raw array bindings on change.
- [ ] Render rows virtualized. Format cells explicitly tracking target currency tags cleanly aligned.
- [ ] Include search bar executing native filtering over grid payloads specifically over (`SKU` or `product_name`).
- [ ] Link standard UI "CSV Export" to the newly configured async API. Ensure the user receives a "Large export started - checking email" toast/snackbar notification rather than pausing the application on the REST connection stream.
- [ ] Gracefully render "Unable to load grid" wrapper if the request payload timeouts under duress (P95 <200ms API breach).

## Execution Directives
- Follow corporate style guides applying specific SCSS grid overrides preventing default gray blocks.
