# Story: SalesInsightsController & AggregationService / FxConversion

## Implementation Scope
Inside the newly decoupled `sales-insights-api` project (Spring Boot 3.1.x), build the REST endpoints returning the DTOs (`SalesAggregationResponse`, `KpiSummaryResponse`). Combine the `FxConversionService` formula with `AggregationService` pre-computations resolving from the materialize view and 364 day offset (LY). Attach Keycloak Security filtering.

## Acceptance Criteria
- [ ] Implement `GET /aggregate` with parameters for `regions`, `currency`, `granularity`.
- [ ] Connect `KeycloakBearerInterceptor` auth-guard verifying users hold active roles.
- [ ] Filter un-authorized regions inside the controller validating against the custom JWT claims securely.
- [ ] Formula for localized value conversion matches architecture specification (divides against base EUR rate, multiplies against target display rate).
- [ ] Provide explicit fallback (`{"fallback": true, "fallback_currency": "EUR", "message": "Rates as of [latest_date]"}`) for unsupported target currencies instead of generic 4xx errors.
- [ ] Telemetry hooked exposing `api_p95_latency_ms`.
