# Story: EcbIngestionTask & Historical Bootstrap

## Implementation Scope
Inside the `i2o-scheduler` codebase, create the XML HTTP client reading ECB endpoints at exactly `16:30 CET`. Add necessary resilience via retry backoffs + SendGrid. 
We must also bootstrap historical LY (Last Year) comparison ranges by importing 13-months of historical XML in a one-off `EcbHistoricalBootstrapTask`.

## Acceptance Criteria
- [ ] Task wired accurately to `SchedulerTaskService`.
- [ ] Executes a GET request against `eurofxref-daily.xml` via standard Spring WebClient/RestTemplate.
- [ ] Schema logic handles >20% spike warnings and rejects malformed inputs entirely.
- [ ] Retries up to 3 times (2s, 4s, 8s backoffs).
- [ ] SendGrid `JavaMailSender` or equivalent alerts the Data Ops group upon all 3 failures.
- [ ] The `EcbHistoricalBootstrapTask` iterates an entire year of data reliably.

## Execution Directives
- `i2o-scheduler/src/main/java/com/corecompete/.../task/EcbIngestionTask.java`
- Avoid duplicating `SendGrid` dependency blocks if standard i2o utilities reside.
