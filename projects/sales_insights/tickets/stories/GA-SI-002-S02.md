# Story: Large CSV Export Background Async Thread 

## Implementation Scope
As dataset requests via the frontend can hit multi-year rows cross 5 regions, the export architecture uses an asynchronous queuing design natively inside Spring. This API drops CSV chunks into Google Cloud Storage and emits a signed-URL access email via SendGrid, neutralizing browser timeout concerns.

## Acceptance Criteria
- [ ] Implement `POST /v1/sales-insights/export` accepting JSON `{"regions":[], "currency": "GBP"}`.
- [ ] Immediate REST response returns `202 Accepted { "job_id": "1234" }` with the background task continuing processing.
- [ ] Executes scalable native Spring `@Async` threading pool explicitly configured in context.
- [ ] Data chunked out of database without blowing VM memory limit to GCS bucket defined uniquely for temp uploads (TTL 24 hours).
- [ ] Generates signed access URL and integrates with i2o platform SendGrid setup (as user notification context "Export Started" and "completed link").

## Execution Directives
- Secure URL payload expiry strictly set to avoid perpetual leak.
