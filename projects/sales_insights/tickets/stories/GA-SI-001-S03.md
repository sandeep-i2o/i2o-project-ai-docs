# Story: CsvIngestionTask & GCS Integration

## Implementation Scope
Extend `i2o-scheduler` to read marketplace blobs from Google Cloud Storage daily after `01:30 CET` extractor deliveries. This bulk-upserts the normalized rows directly into Cloud SQL (`sales_data`). Post-insert, the CRON orchestrates the heavy DB refresh command: `REFRESH MATERIALIZED VIEW CONCURRENTLY`.

## Acceptance Criteria
- [ ] Connects to Data Platform GCS bucket using standard application-default-credentials (managed by GCP).
- [ ] Reads DE, FR, IT, ES, UK region directories dynamically filtering by `date=current` (or parameterized dates for replay).
- [ ] Executes `Bulk INSERT` applying JDBC optimizations over network limits and commits safely on conflict.
- [ ] Executes a Concurrent Refresh statement natively preventing row locks for any active live users.
- [ ] Includes `admin/replay-ingestion` stub or command to backfill partial failures idempotenly.
