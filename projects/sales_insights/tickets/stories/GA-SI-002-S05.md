# Story: ECharts Implementation & KPI Cards

## Implementation Scope
Use the pre-established `frontendapplication-i2oretail` (ECharts 6.0) graphic framework to construct the interactive multi-line trend area chart (`TrendChartComponent`) and the overarching numeric aggregate cards (`KpiCardGroupComponent`). These bind to the `SalesInsightsStateService` mapped streams natively exposing calculated ASP, Revenue, and Units (with Last Year 364 offset delta styling).

## Acceptance Criteria
- [ ] Connect `ngx-echarts` library to the array payload dynamically pushing Region/Category X-Y values seamlessly (< 200ms bounds).
- [ ] Implement hover tooltip properties displaying exact point in time conversion values and raw base.
- [ ] Configure `KpiCardGroupComponent` rendering distinct positive (green) and negative (red) delta percentage flags against LY.
- [ ] Explicitly map empty states handling 0 regions or disconnected/missing network responses preventing library crashes via UI placeholder / `skeleton-loader` classes.
- [ ] Resolve UX edge cases (like `[N/A]` states when previous year data is entirely missing).

## Execution Directives
- Standardize CSS flex-box definitions across components matching UI designs linked in the PRD.
