# Story: SalesInsightsStateService / API wiring & Debouncing (RxJS)

## Implementation Scope
Inside the `frontendapplication-i2oretail` framework, attach `SalesInsightsApiService` implementing `rest-api.service.ts` connecting `/api/v1/sales-insights/aggregate` payload schemas logic. Implement the business requirements (de-bouncing filter changes and caching) using `SalesInsightsStateService` over RxJS BehaviorSubjects. This centralizes state across charts and grids, ensuring only one active multi-region update resolves smoothly.

## Acceptance Criteria
- [ ] Connect `sales-insights-api.service.ts` directly via the Keycloak interceptor token pattern.
- [ ] Process `{"fallback": true, "fallback_currency": "EUR", "message": "Rates as of [latest_date]"}` correctly emitting into a global warning state / notification banner (AR-003 PRD Alignment).
- [ ] Ensure user parameter inputs (currency, regions) apply `.pipe(debounceTime(300), switchMap(...))` to gracefully drop superseded network requests natively in RxJS natively preventing race-conditions.
- [ ] Implement `stale-data-badge` behavior tracking >24 hour freshness flags individually tied to region elements UI array.
- [ ] Expose unified DTO models locally resolving against TS definitions matching API response schema.
